# /continue-dev - Continue Development

Tu dois continuer le développement du projet. Voici comment procéder :

## ÉTAPE 1 : Lire l'état actuel

Lis le fichier `docs/PAUSE-STATE.md` pour connaître :
- Les modules déjà complets (NE PAS REFAIRE)
- La session en cours
- Les tâches restantes
- Les éventuels blocages

## ÉTAPE 2 : Charger le contexte

Selon la session en cours, lis les fichiers de référence :
- `docs/ARCHITECTURE.md` - Structure et stack
- `docs/PRD.md` - Requirements
- `docs/DECISIONS-LOG.md` - Décisions techniques prises
- Les fichiers de dépendances nécessaires

## ÉTAPE 3 : Développer

Pour chaque tâche à faire :
1. Vérifier si le fichier/module existe déjà
2. Créer/modifier le code
3. Mettre à jour `docs/PAUSE-STATE.md` avec le progrès

## ÉTAPE 4 : Mise à jour de l'état

À la fin de la session ou si tu dois t'arrêter :
1. Mettre à jour `docs/PAUSE-STATE.md` avec l'état exact
2. Logger les décisions dans `docs/DECISIONS-LOG.md` si nécessaire
3. Lister les prochaines étapes

## RÈGLES IMPORTANTES

- **Fichiers < 200 lignes** : Si un fichier dépasse, découpe-le
- **1 fichier = 1 responsabilité** : Pas de fichier fourre-tout
- **Toujours mettre à jour PAUSE-STATE.md** : C'est la source de vérité
- **Ne pas deviner** : Si une dépendance manque, la créer d'abord ou signaler le blocage
- **NE JAMAIS refaire un module marqué comme COMPLET**

## COMMANDE

Commence maintenant en lisant `docs/PAUSE-STATE.md` et en continuant le développement.
